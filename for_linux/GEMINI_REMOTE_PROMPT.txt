# Gemini CLI Instruction: Introvert RBN Remote Build & Deploy

You are an expert systems engineer. You are running on a Linux Build Machine. 
Your goal is to compile the Introvert Root Bootstrap Node (RBN) and deploy it to the production server.

## CONTEXT
- Project: Introvert (Sovereign P2P Mesh)
- Target Architecture: Linux x86_64
- Production Server: 47.89.252.80 (Alibaba Cloud)
- Task: Finalize the transition to Port 443 (Messenger-grade bypass).

## YOUR OBJECTIVES

### 1. Build the Hardened Daemon
- Ensure 'build-essential', 'pkg-config', and 'libssl-dev' are installed.
- Ensure the Rust toolchain is installed and set to 'stable'.
- Run: `cargo build --release --bin introvertd`
- The resulting binary will be at: `./target/release/introvertd`

### 2. Deploy to Production
- Use 'scp' to move the binary to the RBN server:
  `scp target/release/introvertd root@47.89.252.80:/opt/introvert/bin/introvertd`
- Use 'scp' to move the service configuration:
  `scp introvertd.service root@47.89.252.80:/etc/systemd/system/introvertd.service`

### 3. Finalize on Production
- SSH into the RBN server to reload and restart:
  `ssh root@47.89.252.80 'chmod +x /opt/introvert/bin/introvertd && systemctl daemon-reload && systemctl restart introvertd'`

## EXECUTION SAFETY
- The user will provide the Alibaba root password manually when prompted by SCP/SSH.
- If the RBN server hangs during any operation, use high-persistence SSH options: 
  `-o IPQoS=none -o ConnectTimeout=60 -o ServerAliveInterval=10`

## VERIFICATION
- After restart, verify the node is listening:
  `ssh root@47.89.252.80 'netstat -tulpn | grep introvertd'`
- It MUST be listening on Port 443 (TCP and UDP).
