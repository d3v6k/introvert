# Fix: Android Background Connectivity & FCM Wake-up Gap

## Problem

When the app is minimized on Android, users see frequent "Connecting" and "Offline" states compared to before. Root causes identified:

### 1. FCM push does NOT wake the Rust engine (CRITICAL)
- `IntrovertFirebaseMessagingService.kt:68-99` starts `IntrovertService` but does NOT call `fetchMailbox()` on the Flutter/Rust side
- The `onWakeup` handler in `alert_service.dart:55-57` is **never invoked** from Android native code
- Push notification shows, but messages aren't actually fetched until user returns to app

### 2. Foreground service has no wake lock for normal users
- `main_shell.dart:268`: `needsStayAwake` is only true for anchor mode or when push is unavailable
- Normal users with FCM get `awake: false` â†’ Android Doze mode suspends network access
- The 15-second `status_check_interval` loop in Rust stops when the process is frozen

### 3. IntroClaw idle mode skips all connection optimization
- `intro_claw.rs:1869-1901`: When `is_background && connected_peers.is_empty()`, only essential maintenance runs
- No connection cycling, no prefetch, no prewarming while backgrounded

### 4. Resume takes 30-90+ seconds to restore connectivity
- 1-second delay before IntroClaw tick fires on resume
- Reconnect ladder: 30s direct â†’ 30s retry â†’ 30s tunnel = up to 90s

---

## Fixes

### Fix 1: Wire FCM push to trigger `fetchMailbox()` on Android
**File:** `android/app/src/main/kotlin/.../IntrovertFirebaseMessagingService.kt`
- After `wakeForegroundService()`, send a MethodChannel call to Flutter: `invokeMethod("onWakeup", null)`
- This triggers `AlertService.onWakeup` â†’ `fetchMailbox()`

**File:** `lib/src/services/alert_service.dart`
- Ensure `onWakeup` handler calls `fetchMailbox()` and triggers an IntroClaw tick

### Fix 2: Always acquire wake lock for the foreground service
**File:** `lib/src/ui/main_shell.dart:266-270`
- Change `needsStayAwake` to always be `true` when backgrounded (not just anchor mode)
- This prevents Doze mode from suspending the network loop
- The foreground service notification already exists; adding a wake lock is minimal overhead

### Fix 3: Run lightweight connection maintenance in idle mode
**File:** `src/intro_claw.rs:1869-1901`
- When `is_background && connected_peers.is_empty()`, still evaluate the `ConnectionStateCycler` once per idle tick (every 5 minutes)
- This ensures the node attempts reconnection periodically even when backgrounded
- Skip expensive operations (prefetch, prewarming) but keep the cycler active

### Fix 4: Reduce resume reconnection time
**File:** `lib/src/ui/main_shell.dart:277-283`
- Remove the 1-second delay before `triggerIntroClawTick()`
- Fire IntroClaw tick immediately on `resumed`
- Also trigger an immediate `forceRefresh()` to re-dial bootstrap nodes
