# Fix: RBN Proactive File-Transfer Topic Subscription

**Date:** 2026-07-14
**Issue:** Cross-network file transfer stalls at 0%. RBN doesn't subscribe to file-transfer gossipsub topics.

---

## Root Cause

The RBN has an auto-subscribe fix that subscribes to `file-transfer-*` topics on first message. But the first message is the `FileChunkRequest` from the receiver, which the RBN can't receive because it's not subscribed to the topic. **Chicken-and-egg problem.**

**Flow:**
1. Sender subscribes to `file-transfer-{id}` topic
2. Sender sends `FileTransfer` manifest via GROUP gossipsub topic
3. RBN receives the manifest (subscribed to group topic)
4. Receiver receives the manifest, calls `startPull`
5. Receiver sends `FileChunkRequest` via `file-transfer-{id}` gossipsub topic
6. **RBN doesn't forward it** — not subscribed to `file-transfer-{id}` topic
7. Sender never receives the request → transfer stalls

## Fix

**File:** `for_linux/src/network/mod.rs` (gossipsub message handler)

When the RBN receives a `[FILE]:` message in a group topic, extract the `transfer_id` and subscribe to `file-transfer-{transfer_id}`.

```rust
// In the gossipsub Event::Message handler:
let topic_str = message.topic.as_str();

// Auto-subscribe to file-transfer topics
if topic_str.starts_with("file-transfer-") {
    let topic = libp2p::gossipsub::IdentTopic::new(topic_str);
    let _ = self.swarm.behaviour_mut().gossipsub.subscribe(&topic);
}

// NEW: If this is a group message containing a [FILE]: manifest,
// subscribe to the corresponding file-transfer topic
if let Ok(payload_str) = std::str::from_utf8(&message.data) {
    if payload_str.contains("[FILE]:") {
        // Extract transfer_id from the JSON payload
        if let Some(start) = payload_str.find("\"transfer_id\":\"") {
            let rest = &payload_str[start + 15..];
            if let Some(end) = rest.find('"') {
                let transfer_id = &rest[..end];
                let ft_topic = libp2p::gossipsub::IdentTopic::new(format!("file-transfer-{}", transfer_id));
                let _ = self.swarm.behaviour_mut().gossipsub.subscribe(&ft_topic);
                info!("[RBN] Subscribed to file-transfer-{} topic from group manifest", transfer_id);
            }
        }
    }
}
```

## Root Cause

The RBN proactive subscription doesn't work because:
1. Sender sends `FileTransfer` manifest via GROUP gossipsub topic (encrypted in GroupAction)
2. RBN forwards raw bytes — doesn't decrypt, can't see `[FILE]:` content
3. Receiver sends `FileChunkRequest` via `file-transfer-{id}` topic
4. RBN NOT subscribed to `file-transfer-{id}` → message not forwarded
5. Sender never receives request → transfer stalls

## Fix: Send FileChunkRequest via Group Topic Fallback

**File:** `src/network/mod.rs` (forward_to_mesh, gossipsub publish section)

When publishing a `FileChunkRequest` to a `file-transfer-{id}` topic, ALSO publish it to the GROUP topic as a fallback. The RBN is subscribed to the group topic and will forward it.

```rust
// In the gossipsub publish section for file payloads:
if let SignalingPayload::FileChunkRequest { ref transfer_id, .. } = payload {
    // Also publish to group topic as fallback — RBN is subscribed to group topics
    if let Some(ref gid) = self.extract_group_id_from_payload(&payload) {
        let group_topic = libp2p::gossipsub::IdentTopic::new(gid);
        if let Ok(bytes) = serde_json::to_vec(&payload) {
            let _ = self.swarm.behaviour_mut().gossipsub.publish(group_topic, bytes);
        }
    }
}
```

**Why this works:** The RBN is subscribed to group topics and forwards messages to all group members. The sender receives the `FileChunkRequest` from the group topic and responds with `FileChunk` via the file-transfer topic (which the receiver is subscribed to).
