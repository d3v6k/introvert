# Intro-Claw UX Overhaul Plan

## Problem
Intro-Claw looks gimmicky in Offline mode вҖ” a chat field that can only do keyword search feels like a broken AI. The engine toggle shows "Not started" even when active. The "Run Maintenance Now" button is redundant since the engine auto-runs.

## Three Changes

### 1. Offline Mode: Query Cards + Floating Result Overlays (NO chat field)
**When AI mode = 0 (Offline):**
- **Hide the text input bar entirely**
- Show a professional grid of tappable query cards instead:
  - Photos/Images, Videos, Files, Contacts, Notes, Messages, Calls
  - Storage Status, Engine Status, Connection Health
- Each card has an icon, label, and brief description
- When tapped вҶ’ execute the query via `processAssistantQuery` вҶ’ show results in a **DraggableScrollableSheet** (floating overlay from bottom)
- The overlay shows the answer text at top + scrollable result list below
- The overlay has a close button and can be dragged to dismiss

**When AI mode = 1 (Hybrid) AND endpoint+key configured:**
- Show the chat input bar (current behavior) вҖ” the LLM can answer naturally
- Keep the suggestion chips above the chat as quick shortcuts
- The query cards are still accessible via a "Quick Actions" button

### 2. Fix Engine Status Display + Remove Maintenance Button
**Engine toggle status fix:**
- The status JSON has `ai_mode`, `api_key_set`, `is_active` but no `tick_count`
- Fix: show "Engine Active" / "Engine Inactive" based on `is_active`, and show tick count from a new `tick_count` field in the status JSON
- Add `tick_count` to `intro_claw_get_status()` FFI response

**Remove "Run Maintenance Now" button:**
- The engine runs automatically on a 5-minute tick loop
- The button adds complexity without value вҖ” remove it from settings

### 3. Header badge shows current mode
- Header badge shows "LOCAL" in Offline mode, "HYBRID" in Hybrid mode
- Badge color changes: green for Offline, amber for Hybrid

## Files to Modify

| File | Changes |
|------|---------|
| `lib/src/ui/assistant_tab.dart` | Major rewrite: query cards grid, floating result overlay, conditional chat/card display based on AI mode |
| `lib/src/ui/main_shell.dart` | Remove "Run Maintenance Now" button, fix "Last tick" display, remove engine active toggle (it auto-runs) |
| `src/lib.rs` | Add `tick_count` to `intro_claw_get_status()` response |
| `src/intro_claw.rs` | Add `tick_count` field to IntroClawService and return it in status |

- Remove the "Run Maintenance Now" button and its `_triggerClawTick` method
- Remove the engine active toggle Switch (engine always auto-runs when `is_active` is true)
- Fix "Last tick" to show actual tick count from status JSON
- Keep the AI mode toggle, endpoint, API key fields

### lib.rs вҖ” Status response

Add `tick_count` field:
```rust
let status = serde_json::json!({
    "ai_mode": mode,
    "api_key_set": has_key,
    "is_active": is_active,
    "tick_count": engine.intro_claw.tick_count,
});
```
