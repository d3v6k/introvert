# Plan: Ensure Nodes Earn At Least 2x of Regular Users

## Problem Analysis

Current test vector comparison:
- **Regular user**: 3,842.6 points → 631.64 $INTR (from 16,438 pool)
- **Edge node (500 $INTR stake)**: 3,842.6 points → 631.64 $INTR (SAME as regular user!)
- **RBN (50,000 $INTR stake)**: 105,861.28 points → 8,700.74 $INTR (13.8x more)

**The issue:** Edge nodes earn the same as regular users. The infra weights (0.01/KB relay, 0.001/sec uptime) contribute only 137.6 points total — negligible compared to social points (3,705).

## Solution: Edge Node Infrastructure Multiplier (30×)

Add `edge_infra_multiplier: 30.0` to `ActivityWeights`. For edge nodes (≥500 $INTR stake, not RBN), infra weights are boosted by 30×:

- RelayBytes: 0.01 × 30 = 0.30/KB
- UptimeSeconds: 0.001 × 30 = 0.030/sec

### Updated Math

**Regular user (no stake):**
- Social: 3,705 points, Infra: 0 → Total: 3,705 → 609.5 $INTR

**Edge node (500 $INTR, 30× multiplier):**
- Social: 3,705, Infra: (51.2×30) + (86.4×30) = 4,128 → Total: 7,833 → 1,287.7 $INTR
- **Ratio: 2.11×** ✓

**RBN (50,000 $INTR, uncapped):**
- Social: 900, Infra: 104,961 → Total: 105,861 → 8,700.74 $INTR
- **Ratio: 14.3×** ✓
